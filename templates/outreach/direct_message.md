# Direct Message Drafts

Use these only for relevant, non-spammy conversations where the recipient has shown interest in GitHub contribution planning, open-source work, or developer tooling.

## Short DM

```text
I noticed you are looking for more focused GitHub contribution ideas. I built a small tool that screens public GitHub issues and filters out risky or low-quality matches. It might help you build a practical weekend contribution plan.

No pressure; the repo is here if useful: REPO_URL
```

## Service Follow-Up

```text
If you want a more tailored version, I offer a small Starter Audit: 10 screened GitHub issue opportunities, top 3 recommended actions, risk notes, and one profile/README positioning suggestion.

It is fixed-scope and does not promise income, merged PRs, or bounty awards.
```

## Boundary Reply

```text
I avoid spam PRs, fake stars/reviews, exploit work, credential handling, scraping behind logins, and platform bypasses. If the request fits a low-risk GitHub automation or contribution workflow, I can scope it clearly.
```
