# Launch Sequence

These drafts are designed for a cautious first launch. Use one channel at a time and only where self-promotion is allowed.

## GitHub Profile Pin

```text
I built GitHub Income Scout to find low-risk GitHub issues worth evaluating: Python automation, CI fixes, docs, GitHub Actions, data scripts, and small tools.

It includes safety filters, Markdown/JSON/CSV exports, a Starter Audit bundle generator, and clear boundaries against spammy or exploit-focused tasks.
```

## Community Post

```text
I released a small open-source tool for developers who want better GitHub contribution leads without spamming maintainers.

GitHub Income Scout scores public issues, filters out high-risk or low-quality work, and exports a short audit report. It is built for Python automation, CI/CD, docs, data scripts, GitHub Actions, README work, and small tools.

Repository: <repo link>

I am looking for feedback on the scoring rules and query packs. I can also prepare a small Starter Audit for developers who want a hand-curated issue shortlist.
```

## First Reply To Interest

```text
Thanks for taking a look. The free repo can run locally and generate its own Markdown/JSON/CSV report.

If you want a hand-curated version, I can prepare a Starter Audit: 5-10 GitHub opportunities matched to your skills, with risk notes, expected effort, acceptance signals, and the first recommended action.

I do not guarantee income, merged PRs, or bounty awards. The goal is to reduce time wasted on unclear or risky issues.
```

## Discounted Starter Audit Offer

```text
For the first few audits, I can do a smaller version at USD 29 while I collect feedback on the format.

Deliverable:
- 5-10 matched GitHub opportunities
- low-risk filter notes
- estimated difficulty and time
- likely acceptance signals
- first action recommendation

I will avoid security exploit work, fake engagement, credential handling, and platform-rule bypass tasks.
```

## Follow-Up After Delivery

```text
I sent the Starter Audit deliverables. The strongest next step is the first item in the report because it has the clearest scope and lowest setup risk.

If you try one of the opportunities, I can also help convert it into a clean branch, test plan, and PR description.
```
