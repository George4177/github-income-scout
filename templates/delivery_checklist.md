# Delivery Checklist: Starter Audit

## Before Research

- Confirm client profile and skill focus
- Confirm low-risk boundaries
- Confirm target languages and task types
- Confirm the client understands this is an opportunity audit, not guaranteed income

## Research

- Run GitHub Income Scout with relevant query pack
- Include safety-filter output
- Remove duplicates
- Open and inspect top candidates manually
- Reject vague, stale, unsafe, or fake-engagement tasks

## Report

The report should include:

- executive summary
- top 10 opportunity table
- top 3 recommended actions
- acceptance criteria for each top action
- failure risks
- estimated time
- profile/README positioning note

## Quality Bar

Do not deliver the audit if:

- all top opportunities are stale or unsafe
- issue scopes are unclear
- the report contains generic advice only
- the client would need to spam maintainers to act on it

## After Delivery

- Ask whether the client wants help with one implementation
- Do not comment, open PRs, claim bounties, or make account-bound commitments without explicit approval
