# GitHub Income Scout Workflow

## Opportunity Report

Use conservative filters and produce a table with:

- project or platform
- link
- task type
- estimated difficulty
- estimated value
- estimated time
- acceptance criteria
- failure risks
- worth-doing decision

## Recommended Query Areas

- Python small tools
- GitHub Actions and CI fixes
- documentation and README work
- data/report automation
- Codex workflow templates
- low-risk help wanted issues

## Rejection Rules

Reject tasks involving:

- security exploitation or privilege escalation
- fake stars, fake reviews, or engagement schemes
- spam comments or mass PRs
- credentials, private tokens, or account secrets
- scraping behind logins
- trading/speculation
- copyright infringement
- payment, API, or platform bypasses

## Starter Audit Deliverable

Prepare:

- executive summary
- top 10 screened opportunities
- top 3 recommended actions
- rejected matches and reasons
- one GitHub profile or README positioning note
- next-step checklist

Avoid promising income, merged PRs, bounty payouts, or sponsorship.

## PR Action Pack

Before implementation:

1. Verify issue is open.
2. Check assignees and open PRs for duplicates.
3. Read README, contribution guide, and test instructions.
4. Draft claim comment only if project norms require it.
5. Prepare scoped implementation, tests, commit message, PR title, and PR body.

Do not post comments or PRs if tool permissions are missing or the task is unsafe.
