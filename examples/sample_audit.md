## Recommended Positioning

Lead with a practical automation angle:

> I build small Python and GitHub workflow tools that save maintainers and solo developers time: issue research, README cleanup, GitHub Actions fixes, and repeatable reporting scripts.

## Starter Action Plan

1. Publish one small utility repository with a clear README and validation command.
2. Submit one low-risk PR only after reading the contribution guide.
3. Pin the utility repository on GitHub with a service-oriented description.
4. Offer a small fixed-scope audit rather than vague "AI automation" work.
